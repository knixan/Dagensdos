# Förklaring av Better Auth i Projektet

Better Auth är ett modernt autentiseringsbibliotek för Next.js som förenklar hantering av användarautentisering. Det delar upp autentiseringen i server-side (backend) och client-side (frontend), med en API-route som brygga mellan dem.

## Översikt över Flödet
Här är en översikt över hur autentiseringen fungerar i detta projekt:

### Backend (Server-Side)
Backend hanterar konfiguration, databasinteraktion och säker logik.

- **Konfiguration (auth.ts)**:
  - Skapar `auth`-instansen med `betterAuth()`.
  - Använder Prisma-adapter för att koppla till PostgreSQL-databasen via Prisma.
    - Lagrar användare, sessioner och konton (se modeller i `schema.prisma`, t.ex. `User`, `Session`, `Account`).
  - Plugins: `admin()` för rollbaserad åtkomst (t.ex. admin-funktionalitet).
  - Email/Password: Aktiverat med validering (minst 8 tecken för lösenord).
    - E-postverifiering är avstängd för nuvarande användare.
  - Felhantering: Enkel logging för felsökning.
  - Miljövariabler: Kräver `BETTER_AUTH_SECRET` för säkerhet.

- **API-Route (route.ts)**:
  - En catch-all-route (`/api/auth/[...all]`) som hanterar alla auth-förfrågningar (GET/POST).
  - Använder `toNextJsHandler(auth)` för att bearbeta requests från klienten (t.ex. sign-in, sign-up).

- **Server-Helpers (server-auth.ts)**:
  - Hjälpfunktioner som `requireAdmin()`, `getSession()` och `requireUser()`.
  - Använder `auth.api.getSession()` för att hämta sessioner från headers.
  - Omdirigerar användaren om de inte är inloggade eller saknar rättigheter (t.ex. admin-roll).

### Frontend (Client-Side)
Frontend hanterar användarinteraktion och synkronisering med servern.

- **Klient-Konfiguration (auth-client.ts)**:
  - Skapar `authClient` med `createAuthClient()` från "better-auth/react".
  - En React-hook-baserad klient som kommunicerar med servern via fetch-requests till API-routen.

- **Hooks och Funktioner**:
  - Exporterar `signIn`, `signUp`, `useSession` för att hantera autentisering i komponenter.

- **Komponenter**:
  - Exempel: `SignInAndProfile.tsx` använder:
    - `useSession` för att visa inloggningsstatus.
    - `authClient.signIn.email()` för inloggning.
    - `authClient.signOut()` för utloggning.
  - Session-data hämtas automatiskt och uppdateras vid förändringar.

### Koppling mellan Backend och Frontend
Autentiseringen kopplas samman genom följande mekanismer:

- **Kommunikation**:
  - Frontend skickar requests (t.ex. sign-in) till `/api/auth/[...all]`.
  - Servern validerar mot databasen och returnerar session-data eller fel.

- **Sessionhantering**:
  - Sessioner lagras som säkra cookies (httpOnly) och i databasen (`Session`-modellen).
  - Klienten får tillgång via `useSession`-hooken, som automatiskt synkroniserar med servern.

- **Databas**:
  - Prisma-modellerna (`User`, `Session`, etc.) används för persistens.
  - Användare har fält som `email`, `role` (för admin), och `emailVerified`.

- **Säkerhet**:
  - Servern hanterar känslig logik (t.ex. lösenordshashning).
  - Klienten skickar bara data; ingen känslig information exponeras på frontend.

## Funktioner som Används
Projektet använder följande Better Auth-funktioner:

- **Sign In/Out**:
  - Email/password-inloggning via `authClient.signIn.email()`.
  - Utloggning via `authClient.signOut()`.
  - Sessioner hanteras automatiskt.

- **Sign Up**:
  - Möjligt via `authClient.signUp.email()`, men inte aktivt använt ännu (kan läggas till senare).

- **Session Management**:
  - `useSession()` för att hämta aktuell användare/session.
  - Server-helpers som `requireUser()` och `requireAdmin()` skyddar routes.

- **Admin-Plugin**:
  - Tillåter rollbaserad åtkomst (t.ex. `role: 'admin'` i `User`-modellen).
  - Används för att begränsa admin-sidor.

- **Email/Password**:
  - Grundläggande autentisering med validering.
  - Password reset är konfigurerat (skickar för närvarande bara console.log; behöver e-postintegration).

- **E-postverifiering**:
  - Avstängd för nuvarande användare, men kan aktiveras senare.

- **Felhantering**:
  - Enkel logging på servern för debugging.

## Sammanfattning
Better Auth förenklar autentisering genom att hantera sessioner, roller och säker kommunikation mellan frontend och backend. För framtida förbättringar, överväg att integrera en riktig e-postleverantör för verifiering och reset-funktioner.
